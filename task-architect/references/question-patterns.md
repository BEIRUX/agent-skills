# Question Patterns by Task Type

## Content Update / Copy Rewrite

1. **Scope**: "I found [N] pages/files that contain [relevant content]. Which section are we focusing on — all of them, or a specific area?"
2. **Current state**: "Here's the current [headline/copy/section]. What's wrong with it — tone, accuracy, outdated info, or full rewrite?"
3. **Voice & tone**: "What tone should the new copy strike? (e.g., confident & direct, warm & approachable, technical & precise)"
4. **Audience**: "Who's reading this — existing clients, prospects, developers, general public?"
5. **Key message**: "What's the single most important thing this page should communicate?"
6. **Constraints**: "Any specific phrases, keywords, or CTAs that must appear? Anything that must NOT appear?"
7. **Reference**: "Is there existing copy elsewhere (another page, a doc, competitor) that captures what you want?"

## New Build / MVP

1. **Core problem**: "In one sentence, what does this solve for the user?"
2. **Users**: "Who uses this — and what's the first thing they do when they open it?"
3. **Stack preference**: "Should this use the existing stack [detected stack], or something different?"
4. **Scope**: "What's the absolute minimum that makes this useful? (MVP boundary)"
5. **Data**: "What data does this need — where does it come from, where does it go?"
6. **Auth**: "Does this need authentication? If yes, what kind?"
7. **Deployment**: "Where does this live — same domain, subdomain, standalone?"

## Bug Fix / Investigation

1. **Symptom**: "What's happening that shouldn't be?"
2. **Expected**: "What should happen instead?"
3. **Reproduction**: "Steps to reproduce, or is it intermittent?"
4. **Scope**: "When did this start — recent deploy, always been there, after a specific change?"
5. **Impact**: "Who's affected — all users, specific role, specific page?"

## Refactor / Improvement

1. **Motivation**: "What's the pain point — performance, readability, maintainability, or prep for a new feature?"
2. **Scope**: "Which files/components are in play? Should I look beyond what's immediately obvious?"
3. **Constraints**: "Any breaking change boundaries — APIs that external things depend on, URLs that can't change?"
4. **Pattern**: "Is there an existing pattern elsewhere in the codebase I should follow?"

## Design / UI Change

1. **Target**: "Which page/component are we changing?"
2. **Reference**: "Do you have a design, screenshot, or reference for what it should look like?"
3. **Responsive**: "Does this need to work differently on mobile vs desktop?"
4. **Interaction**: "Any animations, transitions, or interactive states to consider?"
5. **Consistency**: "Should this match existing components, or is this a new direction?"
